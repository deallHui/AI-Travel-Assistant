# 🔧 手动部署步骤

## 🎯 如果脚本有问题，请按以下步骤手动部署

### 1. 检查环境
```cmd
# 检查Docker是否安装
docker --version

# 检查Docker Compose
docker compose version
```

### 2. 进入Docker目录
```cmd
cd D:\AICD\rag_ai\docker
```

### 3. 创建必要目录
```cmd
# 创建向量数据库目录（如果不存在）
mkdir ..\vectorstores

# 创建日志目录
mkdir logs
```

### 4. 设置环境变量（可选）
```cmd
# 创建.env文件
echo DEEPSEEK_API_KEY=your_api_key_here > .env
```

### 5. 启动服务
```cmd
# 启动API服务
docker compose up -d rag-api

# 或者启动所有服务（包括Nginx）
docker compose up -d
```

### 6. 检查服务状态
```cmd
# 查看容器状态
docker compose ps

# 查看日志
docker compose logs rag-api

# 健康检查
curl http://localhost:8000/health
```

### 7. 测试API
```cmd
# 测试API（需要先设置API密钥）
curl -X POST "http://localhost:8000/api/v1/query" ^
  -H "Authorization: Bearer demo_key_123" ^
  -H "Content-Type: application/json" ^
  -d "{\"question\": \"北京有什么好玩的？\"}"
```

## 🌐 配置内网穿透

### 使用ngrok
```cmd
# 1. 下载ngrok: https://ngrok.com/download
# 2. 注册账号获取token
# 3. 设置token
ngrok config add-authtoken YOUR_TOKEN

# 4. 启动隧道
ngrok http 8000
```

### 使用cpolar
```cmd
# 1. 下载cpolar: https://www.cpolar.com/
# 2. 注册账号
# 3. 启动隧道
cpolar http 8000
```

## 🛠️ 常见问题解决

### 问题1: 端口被占用
```cmd
# 查看端口占用
netstat -ano | findstr :8000

# 修改端口（编辑docker-compose.yml）
ports:
  - "8001:8000"  # 改为8001端口
```

### 问题2: 容器启动失败
```cmd
# 查看详细日志
docker compose logs rag-api

# 重新构建镜像
docker compose build --no-cache rag-api
docker compose up -d rag-api
```

### 问题3: 向量数据库问题
```cmd
# 检查向量数据库目录
dir ..\vectorstores

# 如果为空，需要先初始化RAG系统
cd ..\
python initialize_vectorstore.py
```

## 📋 完整命令列表

### 基本操作
```cmd
# 启动服务
docker compose up -d

# 停止服务
docker compose down

# 重启服务
docker compose restart rag-api

# 查看状态
docker compose ps

# 查看日志
docker compose logs -f rag-api
```

### 维护操作
```cmd
# 进入容器
docker compose exec rag-api bash

# 查看资源使用
docker stats

# 清理无用镜像
docker system prune -f

# 重新构建
docker compose build --no-cache
```

## ✅ 部署成功标志

当您看到以下内容时，说明部署成功：

1. **容器状态正常**
   ```
   docker compose ps
   NAME              IMAGE           STATUS
   rag-travel-api    rag-ai_rag-api  Up 2 minutes
   ```

2. **健康检查通过**
   ```
   curl http://localhost:8000/health
   {"status":"healthy","vectorstore_ready":true}
   ```

3. **API文档可访问**
   - 浏览器打开: http://localhost:8000/docs

4. **内网穿透正常**
   ```
   ngrok http 8000
   Forwarding  https://abc123.ngrok.io -> http://localhost:8000
   ```

## 🎉 部署完成后

1. **记录公网地址**: 如 https://abc123.ngrok.io
2. **配置API密钥**: 在deploy_config.py中设置
3. **测试API功能**: 使用curl或Postman测试
4. **分享给用户**: 提供URL和API密钥
