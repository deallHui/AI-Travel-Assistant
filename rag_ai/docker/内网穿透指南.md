# 🌐 内网穿透部署指南

## 📋 概述

通过内网穿透，您可以将本地Docker服务暴露到公网，让其他人访问您的RAG AI API，无需购买云服务器。

## 🔧 方案对比

| 工具 | 免费额度 | 稳定性 | 速度 | 自定义域名 | 推荐度 |
|------|----------|--------|------|------------|--------|
| ngrok | 1个隧道 | ⭐⭐⭐⭐⭐ | 快 | 付费 | ⭐⭐⭐⭐⭐ |
| cpolar | 2个隧道 | ⭐⭐⭐⭐ | 中等 | 免费 | ⭐⭐⭐⭐ |
| frp | 无限制 | ⭐⭐⭐ | 快 | 需要服务器 | ⭐⭐⭐ |
| 花生壳 | 1个隧道 | ⭐⭐⭐ | 慢 | 付费 | ⭐⭐ |

## 🚀 方案1：ngrok（推荐）

### 安装ngrok
1. 访问 https://ngrok.com/
2. 注册账号并下载ngrok
3. 解压到任意目录

### 配置和使用
```bash
# 1. 设置认证token
ngrok config add-authtoken YOUR_AUTHTOKEN

# 2. 启动Docker服务
cd rag_ai/docker
docker compose up -d rag-api

# 3. 启动ngrok隧道
ngrok http 8000
```

### 高级配置
创建 `ngrok.yml` 配置文件：
```yaml
version: "2"
authtoken: YOUR_AUTHTOKEN
tunnels:
  rag-api:
    proto: http
    addr: 8000
    subdomain: your-rag-api  # 需要付费版本
    inspect: false
  rag-web:
    proto: http
    addr: 80
    subdomain: your-rag-web
```

启动命令：
```bash
ngrok start rag-api
```

## 🔧 方案3：自建frp

### 准备工作
需要一台有公网IP的服务器（可以是最便宜的VPS）

### 服务器端配置
```bash
# 下载frp
wget https://github.com/fatedier/frp/releases/download/v0.52.3/frp_0.52.3_linux_amd64.tar.gz
tar -xzf frp_0.52.3_linux_amd64.tar.gz
cd frp_0.52.3_linux_amd64

# 创建服务器配置
cat > frps.ini << EOF
[common]
bind_port = 7000
dashboard_port = 7500
dashboard_user = admin
dashboard_pwd = your_password
token = your_secret_token
EOF

# 启动服务器
./frps -c frps.ini
```

### 客户端配置
```bash
# 下载Windows版本frp
# 创建客户端配置
cat > frpc.ini << EOF
[common]
server_addr = YOUR_SERVER_IP
server_port = 7000
token = your_secret_token

[rag-api]
type = http
local_ip = 127.0.0.1
local_port = 8000
custom_domains = your-domain.com
EOF

# 启动客户端
frpc.exe -c frpc.ini
```

## 🛠️ Docker + 内网穿透完整部署

### 一键部署脚本
创建 `一键部署.bat`：
```batch
@echo off
echo 🚀 RAG AI 一键部署脚本

:: 启动Docker服务
echo 📦 启动Docker服务...
cd docker
docker compose up -d rag-api

:: 等待服务启动
echo ⏳ 等待服务启动...
timeout /t 10

:: 检查服务状态
curl -s http://localhost:8000/health && (
    echo ✅ API服务启动成功
) || (
    echo ❌ API服务启动失败
    pause
    exit /b 1
)

:: 启动内网穿透
echo 🌐 启动内网穿透...
echo 请选择内网穿透工具:
echo 1. ngrok
echo 2. cpolar
set /p choice="请选择 (1-2): "

if "%choice%"=="1" (
    echo 启动ngrok...
    ngrok http 8000
) else if "%choice%"=="2" (
    echo 启动cpolar...
    cpolar http 8000
) else (
    echo 请手动启动内网穿透工具
)

pause
```

## 🔒 安全配置

### API密钥管理
在 `deploy_config.py` 中设置：
```python
API_KEYS = {
    "free_user_123": {
        "name": "免费用户",
        "rate_limit": "20/hour",
        "permissions": ["query"]
    },
    "premium_user_456": {
        "name": "付费用户",
        "rate_limit": "200/hour", 
        "permissions": ["query", "health"]
    }
}
```

### 访问限制
```python
# 在public_api.py中添加IP白名单
ALLOWED_IPS = [
    "127.0.0.1",
    "YOUR_TRUSTED_IP"
]

@app.middleware("http")
async def ip_whitelist(request: Request, call_next):
    client_ip = request.client.host
    if client_ip not in ALLOWED_IPS:
        return JSONResponse(
            status_code=403,
            content={"error": "IP not allowed"}
        )
    return await call_next(request)
```
