# 🚀 RAG AI系统部署到公网指南

## 📋 部署概述

将您的RAG AI系统部署到公网，让其他人可以通过API调用您的智能旅游助手服务。

## 🛠️ 部署方案选择

### 方案1：云服务器部署（推荐）

#### 推荐云服务商
- **阿里云ECS**: 国内访问速度快，文档丰富
- **腾讯云CVM**: 与微信生态集成好
- **华为云ECS**: AI服务丰富

#### 服务器配置要求
```
最低配置：
- CPU: 2核
- 内存: 4GB

- 带宽: 5Mbps

推荐配置：
- CPU: 4核
- 内存: 8GB
- 存储: 50GB SSD
- 带宽: 10Mbps
```

### 方案2：Docker容器部署

#### 创建Dockerfile
```dockerfile
# 在rag_ai目录下创建Dockerfile
FROM python:3.9-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    gcc \
    g++ \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 复制应用代码
COPY . .

# 暴露端口
EXPOSE 8000

# 启动命令
CMD ["python", "backend/public_api.py"]
```

#### Docker部署命令
```bash
# 构建镜像
docker build -t rag-travel-api .

# 运行容器
docker run -d \
  --name rag-api \
  -p 8000:8000 \
  -e DEPLOY_ENV=production \
  -v $(pwd)/vectorstores:/app/vectorstores \
  rag-travel-api
```

## 📦 部署步骤详解

### 1. 准备服务器环境

#### 连接服务器
```bash
# SSH连接到服务器
ssh root@your-server-ip

# 更新系统
apt update && apt upgrade -y

# 安装必要软件
apt install -y python3 python3-pip git nginx certbot
```

#### 安装Python依赖
```bash
# 安装Python 3.9+
apt install -y python3.9 python3.9-venv python3.9-dev

# 创建虚拟环境
python3.9 -m venv /opt/rag-api
source /opt/rag-api/bin/activate

# 升级pip
pip install --upgrade pip
```

### 2. 部署应用代码

#### 上传代码
```bash
# 方法1: Git克隆（推荐）
cd /opt
git clone https://github.com/your-username/rag-ai.git
cd rag-ai

# 方法2: 直接上传
# 使用scp或其他工具上传代码到服务器
```

#### 安装依赖
```bash
# 激活虚拟环境
source /opt/rag-api/bin/activate

# 安装依赖
pip install -r requirements.txt

# 安装额外的生产环境依赖
pip install uvicorn[standard] gunicorn
```

### 3. 配置环境变量

#### 创建环境配置文件
```bash
# 创建.env文件
cat > /opt/rag-ai/.env << EOF
DEPLOY_ENV=production
DEEPSEEK_API_KEY=your_deepseek_api_key
VECTORSTORE_PATH=/opt/rag-ai/vectorstores
LOG_LEVEL=INFO
EOF
```

#### 设置权限
```bash
chmod 600 /opt/rag-ai/.env
chown -R www-data:www-data /opt/rag-ai
```

### 4. 配置系统服务

#### 创建systemd服务文件
```bash
cat > /etc/systemd/system/rag-api.service << EOF
[Unit]
Description=RAG Travel API Service
After=network.target

[Service]
Type=exec
User=www-data
Group=www-data
WorkingDirectory=/opt/rag-ai
Environment=PATH=/opt/rag-api/bin
ExecStart=/opt/rag-api/bin/python backend/public_api.py
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
EOF
```

#### 启动服务
```bash
# 重载systemd配置
systemctl daemon-reload

# 启动服务
systemctl start rag-api

# 设置开机自启
systemctl enable rag-api

# 检查状态
systemctl status rag-api
```

### 5. 配置Nginx反向代理

#### 创建Nginx配置
```bash
cat > /etc/nginx/sites-available/rag-api << EOF
server {
    listen 80;
    server_name your-domain.com;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host \$host;
        proxy_set_header X-Real-IP \$remote_addr;
        proxy_set_header X-Forwarded-For \$proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto \$scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 60s;
        proxy_read_timeout 60s;
    }
    
    # 静态文件缓存
    location /static/ {
        alias /opt/rag-ai/static/;
        expires 30d;
    }
}
EOF
```

#### 启用配置
```bash
# 启用站点
ln -s /etc/nginx/sites-available/rag-api /etc/nginx/sites-enabled/

# 测试配置
nginx -t

# 重启Nginx
systemctl restart nginx
```

### 6. 配置HTTPS证书

#### 使用Let's Encrypt
```bash
# 安装证书
certbot --nginx -d your-domain.com

# 设置自动续期
crontab -e
# 添加以下行：
# 0 12 * * * /usr/bin/certbot renew --quiet
```

### 7. 配置防火墙

#### UFW防火墙配置
```bash
# 启用防火墙
ufw enable

# 允许SSH
ufw allow ssh

# 允许HTTP和HTTPS
ufw allow 80
ufw allow 443

# 检查状态
ufw status
```

## 🔧 生产环境优化

### 1. 性能优化

#### 使用Gunicorn多进程
```bash
# 创建Gunicorn配置文件
cat > /opt/rag-ai/gunicorn.conf.py << EOF
bind = "127.0.0.1:8000"
workers = 4
worker_class = "uvicorn.workers.UvicornWorker"
worker_connections = 1000
max_requests = 1000
max_requests_jitter = 100
timeout = 60
keepalive = 5
EOF
```

#### 更新systemd服务
```bash
# 修改ExecStart行
ExecStart=/opt/rag-api/bin/gunicorn backend.public_api:app -c gunicorn.conf.py
```

### 2. 监控和日志

#### 配置日志轮转
```bash
cat > /etc/logrotate.d/rag-api << EOF
/opt/rag-ai/logs/*.log {
    daily
    missingok
    rotate 30
    compress
    delaycompress
    notifempty
    create 644 www-data www-data
    postrotate
        systemctl reload rag-api
    endscript
}
EOF
```

#### 监控脚本
```bash
cat > /opt/rag-ai/monitor.sh << EOF
#!/bin/bash
# 检查服务状态
if ! systemctl is-active --quiet rag-api; then
    echo "RAG API服务异常，正在重启..."
    systemctl restart rag-api
    # 发送告警邮件或短信
fi

# 检查磁盘空间
DISK_USAGE=\$(df /opt/rag-ai | tail -1 | awk '{print \$5}' | sed 's/%//')
if [ \$DISK_USAGE -gt 80 ]; then
    echo "磁盘空间不足: \${DISK_USAGE}%"
fi
EOF

chmod +x /opt/rag-ai/monitor.sh

# 添加到crontab
echo "*/5 * * * * /opt/rag-ai/monitor.sh" | crontab -
```

```bash
# 限制直接访问8000端口
iptables -A INPUT -p tcp --dport 8000 -s 127.0.0.1 -j ACCEPT
iptables -A INPUT -p tcp --dport 8000 -j DROP

# 保存规则
iptables-save > /etc/iptables/rules.v4
```

## 📞 故障排除

### 常见问题
1. **服务启动失败**: 检查日志 `journalctl -u rag-api -f`
2. **API无响应**: 检查Nginx配置和防火墙
3. **证书问题**: 重新申请Let's Encrypt证书
4. **性能问题**: 调整Gunicorn worker数量
