# 微信小程序接入RAG AI系统

## 🎯 项目概述

本项目将您的微信小程序与RAG AI智能旅游助手系统成功集成，实现了智能问答功能。用户可以在微信小程序中直接与AI助手对话，获取专业的旅游建议和攻略。

## 🏗️ 系统架构

```
微信小程序前端 → 微信云函数 → RAG AI后端 → 向量数据库 + DeepSeek API
```

## ✨ 主要功能

- 🤖 **智能问答**：基于RAG技术的旅游知识库问答
- 🔍 **AI增强**：当知识库信息不足时，自动调用DeepSeek进行网络搜索
- 📍 **位置感知**：结合用户位置提供精准的本地化建议
- 💬 **友好界面**：现代化的聊天界面，支持消息来源显示和置信度标识
- ⚡ **快捷问题**：预设常见旅游问题，一键发送
- 🔄 **实时状态**：显示AI助手在线状态和系统健康度

## 🚀 快速开始

### 方法一：一键启动（推荐）

1. 双击运行 `启动系统.bat`
2. 脚本会自动检查环境并启动RAG AI后端
3. 按照提示在微信开发者工具中部署云函数

### 方法二：手动启动

1. **启动RAG AI后端**
   ```bash
   cd ..\rag_ai
   python start_backend.py
   ```

2. **安装云函数依赖**
   ```bash
   cd cloudfunctions\getQASystem
   npm install
   ```

3. **在微信开发者工具中**
   - 导入WCDS项目
   - 右键点击 `getQASystem` 云函数
   - 选择"上传并部署：云端安装依赖"

## 🧪 测试验证

### 自动测试
运行连接测试脚本：
```bash
node test_rag_connection.js
```

### 手动测试
1. 在小程序中点击"AI助手"标签
2. 查看状态显示为"AI助手在线"
3. 发送测试问题，如："北京有什么好玩的景点？"
4. 验证AI回复质量和功能完整性

## 🔧 配置说明

### API地址配置
在 `cloudfunctions/getQASystem/index.js` 中修改：
```javascript
const config = {
  development: {
    ragApiBaseUrl: 'http://localhost:8000',  // 开发环境
    timeout: 30000
  },
  production: {
    ragApiBaseUrl: 'https://your-domain.com',  // 生产环境
    timeout: 30000
  }
};
```

### 小程序导航配置
AI助手已添加到底部导航栏，位置在"行程规划"和"天气交通"之间。

## 🔍 故障排除

### 常见问题

1. **"AI助手离线"**
   - 检查RAG AI后端是否运行（http://localhost:8000）
   - 运行 `test_rag_connection.js` 测试连接

2. **云函数调用失败**
   - 确认云函数已正确部署
   - 检查云函数日志是否有错误

3. **回答质量差**
   - 确认向量数据库包含旅游数据
   - 检查DeepSeek API密钥配置

### 调试方法
- 查看微信开发者工具控制台
- 检查云函数运行日志
- 访问 http://localhost:8000/health 检查后端状态
