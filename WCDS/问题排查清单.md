# Index页面无响应问题排查清单

## 🔍 问题现象
- Index页面无响应
- 筛选菜单选择后没有回显
- 目的地列表不显示

## 📋 排查步骤

### 1. 检查云开发环境配置
**检查项目**：
- [ ] 确认 `app.js` 中的云环境ID是否正确
- [ ] 当前配置：`env: "cloud1-5ge75fz18ed0771d"`

**验证方法**：
1. 在微信开发者工具中打开"云开发"控制台
2. 确认环境ID与app.js中的配置一致

### 2. 检查数据库集合和数据
**检查项目**：
- [ ] 确认已创建 `destinations` 集合
- [ ] 确认数据已正确导入（应有6条记录）
- [ ] 检查数据格式是否正确

**验证方法**：
1. 在云开发控制台 → 数据库 → destinations
2. 查看记录数量和数据内容
3. 确认字段名称正确：`id`, `name`, `description`, `budget`, `recommendedTime`, `preferences`

### 3. 检查云函数部署
**检查项目**：
- [ ] 确认 `recommendDestinations` 云函数已部署
- [ ] 检查云函数代码是否正确

**验证方法**：
1. 在云开发控制台 → 云函数
2. 查看 `recommendDestinations` 函数状态
3. 如果没有，需要右键部署云函数

### 4. 查看调试日志
**操作步骤**：
1. 打开微信开发者工具的控制台
2. 刷新index页面
3. 查看以下日志：
   - `index页面加载`
   - `测试云开发连接...`
   - `直接查询数据库成功/失败`
   - `开始加载目的地数据`
   - `云函数返回结果`

### 5. 常见问题和解决方案

#### 问题1：云函数未部署
**现象**：控制台显示"云函数不存在"
**解决**：
1. 右键 `cloudfunctions/recommendDestinations` 文件夹
2. 选择"上传并部署：云端安装依赖"

#### 问题2：数据库权限问题
**现象**：控制台显示权限错误
**解决**：
1. 在云开发控制台 → 数据库 → destinations
2. 点击"权限设置"
3. 设置为"所有用户可读"

#### 问题3：数据格式问题
**现象**：查询成功但数据为空
**解决**：
1. 检查数据库中的字段名是否正确
2. 特别注意：`recommendedTime` 和 `preferences` 字段
3. 重新导入数据

#### 问题4：环境ID错误
**现象**：所有云开发功能都不工作
**解决**：
1. 在云开发控制台查看正确的环境ID
2. 修改 `app.js` 中的环境ID
3. 重新编译项目

## 🔧 临时解决方案

如果云函数有问题，可以临时使用直接数据库查询：

```javascript
// 在 loadDestinations 函数中临时替换
const db = wx.cloud.database()
db.collection('destinations').get().then(res => {
  console.log('直接查询结果：', res)
  this.setData({
    destinations: res.data
  })
}).catch(err => {
  console.error('查询失败：', err)
})
```

这样可以先验证数据库连接和数据是否正常。
