# 🌐 内网穿透解决方案 - 让微信小程序访问本地RAG AI

## 🎯 解决思路

使用内网穿透工具将您本地的 `http://localhost:8000` 暴露到公网，让微信云函数可以访问。

## 🛠️ 推荐工具

### 1. ngrok（国外，稳定）
```bash
# 下载安装 ngrok
# 注册账号获取 authtoken
ngrok authtoken YOUR_AUTHTOKEN
ngrok http 8000
```

### 2. 花生壳（国内，免费）
```bash
# 下载花生壳客户端
# 注册账号，创建内网穿透
# 映射本地 8000 端口
```

### 3. frp（开源，自建）
```bash
# 需要有公网服务器
# 配置 frp 服务端和客户端
```

## 🚀 快速实现步骤

### 步骤1：安装 ngrok
1. 访问 https://ngrok.com/
2. 注册免费账号
3. 下载 Windows 版本
4. 解压到任意目录

### 步骤2：配置 ngrok
```bash
# 在 ngrok 目录下运行
ngrok authtoken YOUR_AUTHTOKEN  # 替换为您的token
ngrok http 8000  # 映射本地8000端口
```

### 步骤3：获取公网地址
ngrok 会显示类似这样的信息：
```
Forwarding  https://abc123.ngrok.io -> http://localhost:8000
```

### 步骤4：更新云函数配置
修改 `cloudfunctions/getQASystem/index.js`：
```javascript
const config = {
  development: {
    ragApiBaseUrl: 'https://abc123.ngrok.io',  // 使用ngrok地址
    timeout: 30000
  }
}

const currentEnv = 'development'  // 切换到开发模式
```

### 步骤5：重新部署云函数
在微信开发者工具中重新部署云函数。

## ⚡ 一键启动脚本

创建 `启动内网穿透.bat`：
```batch
@echo off
echo 🌐 启动内网穿透服务...
echo.
echo 1. 确保RAG AI后端正在运行 (http://localhost:8000)
echo 2. 启动ngrok内网穿透
echo.
cd /d "C:\path\to\ngrok"  # 替换为ngrok实际路径
ngrok http 8000
```

## ⚠️ 注意事项

### 安全考虑
- ngrok免费版地址会变化，每次重启都不同
- 不要在生产环境长期使用内网穿透
- 注意保护API密钥等敏感信息

### 稳定性
- 免费版有连接数限制
- 网络延迟可能较高
- 建议仅用于开发测试

## 🔄 其他方案对比

| 方案 | 优点 | 缺点 | 适用场景 |
|------|------|------|----------|
| 内网穿透 | 快速实现，无需服务器 | 不稳定，有限制 | 开发测试 |
| 云服务器部署 | 稳定可靠，性能好 | 需要成本，配置复杂 | 生产环境 |
| 本地模拟 | 完全离线，响应快 | 功能有限，无真实数据 | 演示展示 |
