# 微信小程序接入RAG AI系统使用说明

## 📋 概述

本文档说明如何将微信小程序与RAG AI智能旅游助手系统进行集成，实现智能问答功能。

## 🏗️ 系统架构

```
微信小程序前端 → 微信云函数 → RAG AI后端 → 向量数据库 + DeepSeek API
```

## 🚀 快速开始

### 1. 启动RAG AI后端服务

首先确保RAG AI后端服务正常运行：

```bash
cd rag_ai
python start_backend.py
```

服务将在 `http://localhost:8000` 启动。

### 2. 配置云函数

#### 2.1 安装云函数依赖

在微信开发者工具中：

1. 右键点击 `cloudfunctions/getQASystem` 文件夹
2. 选择"在终端中打开"
3. 运行以下命令：

```bash
npm install
```

#### 2.2 修改API地址配置

编辑 `cloudfunctions/getQASystem/index.js` 文件中的配置：

```javascript
const config = {
  development: {
    ragApiBaseUrl: 'http://localhost:8000',  // 本地开发地址
    timeout: 30000
  },
  production: {
    ragApiBaseUrl: 'https://your-domain.com',  // 生产环境地址
    timeout: 30000
  }
};
```

### 3. 部署云函数

1. 在微信开发者工具中右键点击 `getQASystem` 云函数
2. 选择"上传并部署：云端安装依赖"
3. 等待部署完成

### 4. 测试功能

1. 在微信开发者工具中打开小程序
2. 导航到"AI对话"页面
3. 输入旅游相关问题进行测试

## 🔧 配置说明

### API配置文件

`config/api.js` 文件包含了API相关配置：

- `ragApiBaseUrl`: RAG AI后端服务地址
- `timeout`: 请求超时时间
- `endpoints`: API端点配置

### 云函数配置

`cloudfunctions/getQASystem/index.js` 主要功能：

- 接收小程序的查询请求
- 转发请求到RAG AI后端
- 处理响应和错误
- 返回格式化的结果

## 🛠️ 开发调试

### 本地调试

1. 确保RAG AI后端服务运行在 `http://localhost:8000`
2. 在微信开发者工具中启用"不校验合法域名"
3. 测试云函数调用

### 生产部署

1. 将RAG AI后端部署到服务器
2. 配置HTTPS域名
3. 在微信小程序后台添加服务器域名到request合法域名
4. 更新云函数中的生产环境配置

## 🔍 故障排除

### 常见问题

1. **云函数调用失败**
   - 检查云函数是否正确部署
   - 确认依赖包是否安装完成

2. **RAG AI连接失败**
   - 检查后端服务是否运行
   - 确认网络连接是否正常
   - 验证API地址配置是否正确

3. **请求超时**
   - 检查网络状况
   - 适当增加timeout配置
   - 确认服务器响应时间

### 调试方法

1. 查看微信开发者工具控制台日志
2. 检查云函数日志
3. 测试RAG AI后端健康状态：`GET /wechat/health`

## 📊 API接口说明

### 微信小程序专用接口

#### POST /wechat/query
查询RAG AI系统

**请求参数：**
```json
{
  "question": "用户问题",
  "openid": "微信用户ID",
  "location": {
    "latitude": 纬度,
    "longitude": 经度,
    "city": "城市名称"
  },
  "top_k": 3
}
```

**响应格式：**
```json
{
  "success": true,
  "message": "查询成功",
  "data": {
    "answer": "AI回答内容",
    "sources": ["信息来源"],
    "confidence": 0.9,
    "enhanced_with_ai": false
  },
  "timestamp": "2024-01-01T00:00:00Z"
}
```

#### GET /wechat/health
健康检查接口
