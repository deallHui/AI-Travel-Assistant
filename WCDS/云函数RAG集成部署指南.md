# 微信小程序云函数RAG AI集成部署指南

## 🎯 问题解决

您的微信小程序AI助手总是返回固定回答的原因是使用了测试版本的云函数。现在已经更新为真正连接RAG AI系统的版本。

## 📋 部署步骤

### 1. 安装云函数依赖

在微信开发者工具中：

1. 右键点击 `cloudfunctions/getQASystem` 文件夹
2. 选择"在终端中打开"
3. 运行以下命令：

```bash
npm install
```

这将安装新增的 `axios` 依赖，用于调用RAG AI后端接口。

### 2. 确认RAG AI后端运行

确保您的RAG AI后端服务正在运行：

```bash
cd rag_ai
python start_backend.py
```

服务应该在 `http://localhost:8000` 运行。

### 3. 部署云函数

1. 在微信开发者工具中右键点击 `getQASystem` 云函数
2. 选择"上传并部署：云端安装依赖"
3. 等待部署完成

### 4. 测试功能

1. 在微信开发者工具中打开小程序
2. 导航到"AI对话"页面
3. 输入旅游相关问题，例如：
   - "推荐一些北京的必去景点"
   - "三亚有什么好吃的美食"
   - "上海到杭州怎么去最方便"

## 📊 云函数更新内容

### 主要变更

1. **真正的RAG集成**：
   - 调用 `/wechat/query` 接口获取智能回答
   - 支持位置信息增强查询
   - 返回真实的知识库搜索结果

2. **健康检查功能**：
   - 支持 `action: 'health'` 参数
   - 检查RAG AI系统运行状态
   - 返回系统详细信息

3. **错误处理优化**：
   - 网络连接失败时的降级处理
   - 友好的错误提示信息
   - 详细的日志记录

### 配置说明

云函数支持开发和生产环境配置：

```javascript
const config = {
  development: {
    ragApiBaseUrl: 'http://localhost:8000',  // 本地开发
    timeout: 30000
  },
  production: {
    ragApiBaseUrl: 'https://your-domain.com',  // 生产环境
    timeout: 30000
  }
}
```

## 🔍 故障排除

### 常见问题

1. **云函数部署失败**
   ```
   解决方案：
   - 确保 package.json 包含 axios 依赖
   - 重新运行 npm install
   - 检查网络连接
   ```

2. **RAG AI连接失败**
   ```
   解决方案：
   - 确认后端服务运行在 http://localhost:8000
   - 检查防火墙设置
   - 验证健康检查接口：GET /wechat/health
   ```

3. **回答质量不佳**
   ```
   解决方案：
   - 检查向量数据库是否包含相关旅游数据
   - 确认DeepSeek API密钥配置正确
   - 查看后端日志了解详细信息
   ```

### 调试方法

1. **查看云函数日志**：
   - 在微信开发者工具中查看云函数调用日志
   - 检查请求和响应数据

2. **测试后端接口**：
   ```bash
   # 健康检查
   curl http://localhost:8000/wechat/health
   
   # 测试查询
   curl -X POST http://localhost:8000/wechat/query \
     -H "Content-Type: application/json" \
     -d '{"question":"推荐北京景点","openid":"test"}'
   ```

3. **检查系统状态**：
   - 运行 `检查系统状态.bat` 脚本
   - 查看各组件运行状态
